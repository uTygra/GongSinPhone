01 · 사업 전제

무엇을 만드는가

단말을 제조하지 않습니다. 부모가 집에 있는 삼성 갤럭시를 우리 서비스에 등록하면, QR 스캔 한 번으로 폰이 공신폰으로 바뀝니다. 차단 상태를 유지·관리하는 대가로 월 구독료를 받는 SaaS입니다.

핵심 명제 공신폰의 본질은 하드웨어가 아니라 "차단이 살아있는 상태"입니다. 그 상태를 OS가 보증하는 수준으로 유지하고, 부모가 원격으로 관리할 수 있게 하는 것이 상품입니다.

왜 삼성 갤럭시인가

이유 01

Knox = 무료 하드웨어 보안

KPE Premium 라이선스가 2021년부터 무료. ARM TrustZone 기반 하드웨어 봉쇄로 세이프모드·소프트웨어 업데이트까지 잠글 수 있습니다.

이유 02

한국 BYOD 풀이 가장 크다

국내 스마트폰 시장 점유율 1위. 집에 있는 공기계 = 대부분 갤럭시. 타깃 고객의 보유폰이 곧 재고입니다.

이유 03

AMAPI와 완전 호환

Google AMAPI가 갤럭시에서 QR 등록·키오스크·정책 PATCH 모두 정식 지원. Knox Mobile Enrollment로 추후 대량 자동화도 가능.

① 엔터프라이즈 + 정책 생성 (가입 시 1회)

// lib/amapi.ts
import { google } from 'googleapis'

const androidmanagement = google.androidmanagement('v1')

export async function createEnterpriseForUser(userId: string) {
  // 1. 서비스 계정 인증
  const auth = new google.auth.GoogleAuth({
    keyFile: process.env.GOOGLE_SERVICE_ACCOUNT_KEY,
    scopes: ['https://www.googleapis.com/auth/androidmanagement'],
  })

  // 2. 엔터프라이즈 생성 (사용자 1명 = 엔터프라이즈 1개)
  const enterprise = await androidmanagement.enterprises.create({
    projectId: process.env.GOOGLE_PROJECT_ID,
    auth,
    requestBody: {
      enterpriseDisplayName: `gongshin-user-${userId}`,
      pubsubTopic: `projects/${process.env.GOOGLE_PROJECT_ID}/topics/amapi-events`,
    },
  })
  const enterpriseName = enterprise.data.name  // enterprises/{id}

  // 3. 공부 정책 생성
  await androidmanagement.enterprises.policies.patch({
    auth,
    name: `${enterpriseName}/policies/study`,
    updateMask: 'applications,kioskCustomLauncherEnabled,kioskCustomization,developerSettings,safeBootDisabled,factoryResetDisabled',
    requestBody: buildStudyPolicy(enterpriseName),
  })

  // 자유·만료 정책도 동일하게 생성
  return enterpriseName
}

export async function createEnrollmentToken(enterpriseName: string) {
  const res = await androidmanagement.enterprises.enrollmentTokens.create({
    auth,
    parent: enterpriseName,
    requestBody: {
      policyName: `${enterpriseName}/policies/study`,
      duration: '3600s',  // 1시간 유효 (부모가 그 시간 안에 스캔)
      allowPersonalUsage: 'PERSONAL_USAGE_DISALLOWED',  // 개인 사용 차단
    },
  })
  // res.data.value → QR로 인코딩할 JSON 문자열
  return res.data.value  // 이 값을 QR 이미지로 변환해 화면에 표시
}

③ 정책 즉시 변경 (콘솔에서 앱 토글)

// app/api/devices/[id]/policy/route.ts
export async function PATCH(req: Request, { params }) {
  const { allowedApps } = await req.json()
  const device = await prisma.device.findUnique({ where: { id: params.id } })

  // 현재 정책(study)을 조회 후 applications 배열만 교체
  const updatedPolicy = buildStudyPolicy(device.amapiPolicyName, allowedApps)

  await androidmanagement.enterprises.policies.patch({
    auth,
    name: device.amapiPolicyName,
    updateMask: 'applications',  // applications만 업데이트
    requestBody: updatedPolicy,
  })
  // → Pub/Sub으로 단말에 실시간 반영 (30초~2분 내)
  return Response.json({ ok: true })
}

④ 스케줄 cron (시간표 자동 교체)

// app/api/schedule/apply/route.ts  ← vercel.json cron: "*/15 * * * *"
export async function POST() {
  const now = new Date()
  const devices = await prisma.device.findMany({
    where: { status: 'ACTIVE' },
    include: { schedule: true, user: { include: { subscription: true } } }
  })

  for (const device of devices) {
    if (device.user.subscription?.status !== 'ACTIVE') continue

    const isStudyTime = checkSchedule(device.schedule, now)
    const targetPolicy = isStudyTime ? 'study' : 'free'
    const policyBase = device.amapiPolicyName.split('/').slice(0,-1).join('/')

    await androidmanagement.enterprises.devices.patch({
      auth,
      name: device.amapiDeviceId,
      updateMask: 'policyName',
      requestBody: { policyName: `${policyBase}/${targetPolicy}` }
    })
  }
  return Response.json({ applied: devices.length })
}

⑤ 구독 만료 처리

// 포트원 webhook → 결제 실패 시
export async function handlePaymentFailed(userId: string) {
  const devices = await prisma.device.findMany({ where: { userId, status: 'ACTIVE' } })

  for (const device of devices) {
    const policyBase = device.amapiPolicyName.split('/').slice(0,-1).join('/')
    // 만료 정책으로 전환 → 전화만 되고 갱신 화면 표시
    await androidmanagement.enterprises.devices.patch({
      auth,
      name: device.amapiDeviceId,
      updateMask: 'policyName',
      requestBody: { policyName: `${policyBase}/expired` }
    })
    await prisma.device.update({ where: { id: device.id }, data: { status: 'EXPIRED' } })
  }
}

운영 자동화 목표

BYOD 모델의 핵심은 택배 입고 없이 셀프로 끝나는 것입니다. 1인 운영 기준 대응 한계를 아래와 같이 설계합니다.

결제 흐름 (포트원 v2)

// 1. 카드 등록 (빌링키 발급)
POST /api/billing/subscribe
→ 포트원 결제창 → 빌링키 발급 → DB 저장

// 2. 매월 자동 결제 (서버 cron)
POST /api/billing/charge  ← 매월 1일 오전 9시
→ 포트원 빌링키로 결제
→ 성공: subscription.currentPeriodEnd += 30일
→ 실패: failCount++ → 3일 후 재시도
→ 3회 실패: handlePaymentFailed() → 만료 정책 적용

// 3. 갱신 후 복원
POST /api/billing/webhook  ← 포트원 이벤트
→ payment.paid → 만료 단말 모두 study 정책으로 복원
